# Roadmap

## Near Term

- Add repo-level release discipline and installation documentation.
- Strengthen README landing-page clarity and trust signals.
- Keep the bilingual single-file documentation polished and easy to navigate.
- Expand evaluation checklists for regression-style manual validation.

## Next Stage

- Add more domain-specific guidance to `references/domain-routing.md`.
- Add more real examples across NLP, CV, multimodal, RL, and systems tasks.
- Add a lightweight compatibility note for Codex environments and local installation behavior.
- Add machine-readable output examples such as JSON sidecars for baseline decisions.

## Longer Term

- Add richer testcases for output completeness and rule compliance.
- Add more formal release notes and version-to-feature mapping.
- Consider optional plugin-style packaging only if it improves installation without adding friction.
- Consider lightweight helper scripts only when they reduce manual work without introducing API setup burden.

## Non-Goals for Now

- No mandatory external API configuration.
- No provider-specific hosted backend.
- No heavy automation layer that makes the skill harder to inspect or maintain.
